# BUGS REAIS IDENTIFICADOS

---

# BUG 1 — LASER EXECUTANDO LÓGICA 3 VEZES

Arquivo:

```
systems/laser.py
```

Problema:

Blocos duplicados:

```
if gs.laser_charging
if gs.laser_active
```

aparecem **3 vezes no arquivo**.

Consequências:

* timers decrementam múltiplas vezes
* comportamento inconsistente
* bugs de balanceamento

Correção:

Manter apenas **1 execução de cada bloco**.

---

# BUG 2 — ACCURACY DO MINIBOSS NÃO FUNCIONA

Arquivo:

```
entities/miniboss.py
```

Função:

```
get_fight_stats()
```

Retorna:

```
accuracy = 0
```

Problema:

```
shots_taken nunca é atualizado
```

Resultado:

```
accuracy sempre 0
```

Correção:

Adicionar:

```
shots_taken
shots_hit
```

e calcular accuracy corretamente.

---

# BUG 3 — NEAR MISS NÃO REGISTRA NO MINIBOSS

Arquivo:

```
systems/dodge.py
```

Problema:

Near miss executa:

```
log_near_miss()
```

Mas não atualiza:

```
gs.miniboss.near_miss
```

Correção:

Adicionar atualização do contador.

---

# BUG 4 — FAILSAFE INADEQUADO

Arquivo:

```
main.py
```

Existe um failsafe:

```
stage_duration * 4
```

Problemas:

* lógica de gameplay no main loop
* dependência implícita de duração de evento
* interfere em boss events

Correção:

Remover failsafe do `main.py`.

Run termination deve ser responsabilidade do:

```
run_director
```

# CONFLITOS DE AUTORIDADE

---

# AUTHORITY 1 — MORTE DE INIMIGOS

Atualmente inimigos podem morrer por:

```
bullet_enemy_collision
rage_system
```

Mas às vezes inimigos são removidos diretamente:

```
gs.enemies.remove(enemy)
```

Correção:

A morte deve ocorrer **apenas via**:

```
kill_enemy()
```

---

# AUTHORITY 2 — SPAWN

Spawn ocorre em:

```
spawn.py
run_events.py
boss_logic.py
```

Correção:

Centralizar spawn em:

```
systems/spawn.py
```

Outros sistemas devem **solicitar spawn**.

---

# AUTHORITY 3 — TERMINAÇÃO DE RUN

A run termina em:

```
main.py

player_enemy_collision
```

Correção:

deve finalizar runs.
